--------------------------------------------------
INFRASTRUCTURE REQUIREMENTS
--------------------------------------------------

- Platform: Google Cloud VM (Ubuntu LTS)
- Containerization: Docker + Docker Compose
- Backend: Node.js (Next.js 14) + FastAPI or Express services
- Database: PostgreSQL (with pgvector for AI memory)
- Cache/Queue: Redis
- Reverse Proxy: Nginx
- SSL: Certbot HTTPS
- CI/CD: GitHub Actions
- Dev Interface: Replit (used safely, no secrets stored there)

--------------------------------------------------
SYSTEM ARCHITECTURE
--------------------------------------------------

Design the system as modular services:

1. AI ROUTER SERVICE

   - Supports:
     - OpenAI
     - Anthropic Claude
     - Google Gemini
     - Optional local model (Ollama)
   - Routes based on:
     - cost
     - speed
     - task type (coding, marketing, research, etc.)
   - Includes fallback logic

2. AGENT TASK ENGINE
   - Queue-based system using Redis
   - Can:
     - generate content
     - send emails
     - schedule posts
     - execute workflows
   - Supports automation triggers

3. MEMORY SYSTEM
   - Stores:
     - conversations
     - tasks
     - project data
   - Uses embeddings (pgvector)

4. CONNECTOR LAYER
   - Build modular connectors for:
     - GitHub
     - Google Drive
     - Gmail
     - Google Calendar
     - Stripe
     - YouTube API
     - Facebook / Instagram
     - Shopify
   - Store ONLY metadata in DB
   - Store secrets in environment variables or secret manager

5. DIY VISION MODULE
   - Input: image / text / voice
   - Output:
     - step-by-step plan
     - materials list
     - tutorials (YouTube)
     - pro recommendations
   - Integrate Stripe for:
     - subscriptions
     - paid tutorials
     - pro calls

6. AUTONOMOUS MARKETING ENGINE
   - Generates:
     - social posts
     - email campaigns
     - video scripts
   - Saves drafts
   - Requires admin approval before publishing

7. ADMIN DASHBOARD
   - View:
     - AI usage
     - API costs
     - connectors status
     - tasks
     - campaigns
   - Manual approve/reject actions
